# CompTIA CySA+ CS0-003 Study Guide

A comprehensive study guide for the CompTIA CySA+ (Cybersecurity Analyst) CS0-003 certification exam, synthesized from course materials and official exam objectives. Built as a single dark-mode HTML file with full anchor navigation.

🔗 **[View the Study Guide](https://conwilso92.github.io/cysa-plus-study-guide)**

## Coverage

All 13 chapters mapped to the 4 official exam domains:

| Domain | Weight | Chapters |
|---|---|---|
| 1.0 Security Operations | 33% | Ch. 01, 02, 03, 04, 05 |
| 2.0 Vulnerability Management | 30% | Ch. 05, 06, 07, 08 |
| 3.0 Incident Response and Management | 20% | Ch. 09, 10, 11, 13 |
| 4.0 Reporting and Communication | 17% | Ch. 12, 13 |

### What's Inside

- **Chapter 01 — Today's Cybersecurity Analyst:** Privacy vs. security distinction, GAPP 10 principles (management through monitoring/enforcement), NIST SP 800-30 risk assessment (threat/vulnerability/risk definitions), threat types (adversarial/accidental/structural/environmental), qualitative vs. quantitative risk assessment (qualitative = testable on exam), risk matrix (probability × magnitude), NAC and 802.1X (supplicant/authenticator/authentication server roles), RADIUS, firewall types (packet filtering/stateful/NGFW/WAF/triple-homed), honeypots vs. honeynets vs. honeytokens, DNS sinkholes, network segmentation, endpoint hardening, Group Policy enforcement, penetration test phases (planning/attack including rules of engagement: timing/scope/authorization), reverse engineering as static analysis, machine learning in security, efficiency and process improvement principles, single pane of glass, API/webhook/plugin tool integration
- **Chapter 02 — System and Network Architecture:** Cloud service models (SaaS/PaaS/IaaS with customer vs. provider responsibility), cloud deployment models (public/private/community/hybrid), shared responsibility model and misconfiguration risk, Zero Trust (never trust, always verify), SASE (SD-WAN + cloud security convergence), SDN (software-defined networking), network segmentation and microsegmentation, IaC benefits and security risks (hardcoded credentials, version control exposure), MFA, SSO, federation (SAML/OAuth/OpenID Connect), PAM (privilege vaulting, JIT access, PAWs), passwordless authentication, CASB (inline vs. API-based), Windows Registry root keys (HKLM/HKCU/HKCR/HKU/HKCC), system hardening practices, file structure and config locations (Windows/Linux/macOS), hardware architecture impact on malware, log ingestion and NTP time synchronization, logging levels, serverless/FaaS, virtualization vs. containerization, PKI and certificate lifecycle, SSL/TLS inspection, DLP, PII, CHD
- **Chapter 03 — Malicious Activity:** Network indicators (beaconing/bandwidth consumption/irregular P2P/rogue devices/scans-sweeps/unusual spikes/unexpected ports), host indicators (processor/memory/drive/unauthorized software/malicious processes/unauthorized changes/unauthorized privileges/data exfiltration/filesystem anomalies/registry run key changes/unauthorized scheduled tasks), DoS and DDoS patterns and cloud-based mitigation, SIEM (Splunk/Graylog/ELK), SOAR (playbooks/automation/APIs), EDR, XDR, Wireshark filter syntax, tcpdump command examples, WHOIS, AbuseIPDB, VirusTotal, Strings, Joe Sandbox, Cuckoo Sandbox, email header analysis, SPF/DKIM/DMARC (authentication/signing/policy enforcement), phishing indicators, active monitoring (ping/iPerf), passive monitoring, router-based monitoring, NetFlow/IPFIX, SNMP (ports 161/162), programming languages in malicious activity detection (JSON/XML/Python/PowerShell/shell/regex)
- **Chapter 04 — Threat Intelligence:** Intelligence cycle (requirements → collection → processing/analysis → dissemination → feedback), confidence levels (timeliness/relevancy/accuracy), threat actors (APT/nation-state/organized crime/hacktivists/script kiddie/insider-malicious-unintentional/supply chain), TTPs and MITRE ATT&CK framework, open source intelligence (social media/blogs/government bulletins/CERT/CSIRT/dark web), closed source intelligence (paid feeds/ISACs/internal sources), uses of threat intelligence (IR/vulnerability management/risk management/security engineering/detection), proactive threat hunting (hypothesis-based, focus areas: misconfigurations/isolated networks/critical assets), IoC collection/analysis/application framework, common IoC types, active defense and tarpits, honeypots for intelligence gathering, three levels of threat intelligence
- **Chapter 05 — Reconnaissance & Intelligence Gathering:** Passive reconnaissance (OSINT sources, log and configuration analysis), active reconnaissance tools (nmap — scan modes/OS fingerprint/service detection, Angry IP Scanner — Java requirement/fetcher modules/port configuration, Maltego — link analysis/transforms, Metasploit Framework — scanning/enumeration/exploitation, Recon-ng — modular OSINT aggregation, hping — custom packet crafting, Zenmap — topology visualization), port ranges (well-known 0–1023, registered 1024–49151, dynamic 49152–65535), service and version identification (banner grabbing), OS fingerprinting, enumeration (OS details/usernames/shares/directories/web server info), asset discovery and device fingerprinting
- **Chapter 06 — Designing a Vulnerability Management Program:** Active vs. passive scanning trade-offs, credentialed vs. non-credentialed scanning (accuracy/false positive rates), agent-based vs. agentless, internal vs. external perspectives, static vs. dynamic analysis (reverse engineering/fuzzing), scheduling and sensitivity level considerations, segmentation impact on scans, regulatory requirements (PCI DSS quarterly/after changes, FISMA NIST 800-53), OT/ICS/SCADA special handling, plugin management (daily updates/disable unnecessary), vulnerability management lifecycle (discovery → scan → prioritize → remediate → verify → report), Nessus configuration/templates/automated reporting, OpenVAS, Qualys dashboard, Nikto, Arachni, OWASP ZAP, Burp Suite, Scout Suite/Prowler/Pacu cloud assessment tools, compliance frameworks (PCI DSS requirements 11.2/11.2.1/11.2.2, FISMA/NIST SP 800-53, CIS benchmarks, OWASP, ISO 27000 series — differences and purposes)
- **Chapter 07 — Vulnerability Management:** Full CVSS v3 metric matrix (Attack Vector/Complexity/Privileges Required/User Interaction/Scope/Confidentiality/Integrity/Availability — all options and meanings), CVSS vector string interpretation (example: AV:N/AC:L/PR:N/UI:N/S:U/C:H/I:N/A:N), CVSS score calculation (ISS formula, Impact Score by Scope, Exploitability Score, Base Score rules — if Impact=0 then Base=0), true/false positive/negative validation, context awareness (internal/external/isolated), exploitability and weaponization (CISA KEV catalog), asset value consideration, zero-day risk, vulnerability types (buffer overflow — stack/heap/integer/stack, privilege escalation including Dirty COW, arbitrary code execution, missing patches — Verizon DBIR data, EOL systems, insecure protocols, misconfiguration, SQL injection, XSS — persistent/reflected/DOM, authentication vulnerabilities including credential stuffing, certificate issues — name mismatch/expiry/unknown CA, IoT/ICS/embedded systems including RTOS/SoC/FPGA/CAN bus, LFI/RFI, SSRF, broken access control, CSRF, directory traversal, insecure design, cryptographic failures)
- **Chapter 08 — Analyzing Vulnerability Scans:** Risk identification, risk calculation (severity = probability × magnitude), risk management strategies (avoidance/transference/acceptance/mitigation), ERM, BIA (critical functions/RTO/RPO/MTBF/MTTR), security control categories (managerial/operational/technical) and types (preventative/detective/responsive/corrective/compensating), threat modeling elements (adversary capability/attack surface/attack vectors/impact/likelihood), attack surface management (edge discovery/passive discovery/security controls testing/pen testing/bug bounty/attack surface reduction), configuration management vs. change management, baselining, version control, patch management (test → deploy → validate → rollback), SDLC models (waterfall/spiral/agile/RAD), DevOps vs. DevSecOps, CI/CD pipeline security risks, secure coding best practices (input validation/output encoding/session management/authentication/data protection/parameterized queries), code analysis types (SAST/DAST/fuzzing/fault injection/mutation testing/regression/UAT/debuggers including Immunity Debugger and GDB), policy hierarchy (policy → standards → procedures → guidelines), exceptions and compensating controls documentation
- **Chapter 09 — Building an Incident Response Program:** Cyber Kill Chain all 7 phases with descriptions (Reconnaissance → Weaponization → Delivery → Exploitation → Installation → C2 → Actions on Objectives), Diamond Model (adversary/victim/infrastructure/capability — following edges to discover additional information), MITRE ATT&CK (tactics as columns/techniques as rows/sub-techniques), OSSTMM, OWASP Testing Guide, CSIRT composition and scope, coordination and information sharing, incident classification (functional impact/economic impact/recoverability/information impact), IR phases (preparation/detection-analysis/containment/eradication/recovery/post-incident — NIST model), preparation phase (IR plan/playbooks/tabletop exercises/tools/training/BC-DR), NIST IR toolkit (digital forensic workstation/backup devices/laptops/spare server-network equipment/blank removable media/portable printer/forensic and packet capture software/bootable USB/office supplies and evidence materials), detection and analysis (alerts/logs/public information/people), actions to improve analysis effectiveness (profile/baseline/logging policy/event correlation/clock sync/knowledge base/traffic capture/filter/escalation)
- **Chapter 10 — Incident Detection and Analysis:** IoC definition and three lenses (collection/analysis/application), IoC feed content types (IP addresses and hostnames/domain names/file hashes/behavior-based TTPs/email addresses/URL patterns), network-based IoCs (unusual outbound traffic/unexpected protocol use/DNS anomalies/traffic timing), host-based IoCs (resource anomalies/registry run keys/new accounts/scheduled tasks/new software/file system changes), user and account behavior IoCs (privilege escalation/impossible travel/bot-like patterns/group membership changes), DNS-based IoCs (recently registered domains/DGA patterns/high-volume queries), file and configuration IoCs, DoS IoCs, evidence acquisition and preservation (chain of custody/data integrity via hashing/preservation with write protection/legal holds and implications of non-compliance)
- **Chapter 11 — Containment, Eradication & Recovery:** Scope and impact assessment before containment, containment strategies (proactive network segmentation/isolation — keep powered on for forensics/removal — take offline/remediation/compensating controls), isolation vs. removal distinction (isolation = cut network but stay on; removal = full offline), attacker identification challenges per NIST guidance, law enforcement engagement considerations, eradication (reconstruction and re-imaging from known-good baseline — not from potentially compromised backups, address root cause first, patch before returning to service), sanitization and disposal methods (overwrite/degauss/physical destruction — method selection based on data sensitivity and media type), recovery validation (rescan/monitor/confirm no backdoors/validate data integrity), post-incident activities, final report components (chronology/root cause/evidence/actions taken with rationale/impact estimates/post-recovery validation/lessons learned issues)
- **Chapter 12 — Reporting and Communication:** Vulnerability report critical elements (vulnerabilities/affected hosts with hostname-IP/risk score/mitigation/recurrence/prioritization), compliance reports, action plans (configuration management/patching/compensating controls/awareness training/timeline), stakeholder types (technical/security-audit-compliance/security management/leadership-executive — different content for each audience), management dashboards vs. technical reports, inhibitors to remediation (MOU/SLA/organizational governance/business process interruption/degraded functionality/legacy systems/proprietary systems), vulnerability metrics (trends/top 10/critical and zero-days/SLO compliance), incident response communication stakeholders, incident declaration and escalation, IR report content (executive summary/who-what-when-where-why/recommendations/timeline/impact/scope/evidence), communications management (legal counsel/public relations — customer and media/regulatory reporting with specific timelines: GDPR 72hr/HIPAA 60 days/PCI DSS immediate/law enforcement), root cause analysis 4-step process, lessons learned process, KPIs (MTTD/MTTR/mean time to remediate/alert volume — definitions and interpretation)
- **Chapter 13 — Performing Forensic Analysis:** Order of volatility table (CPU registers → RAM → network connections → running processes → disk → local logs → remote logs → physical configuration), forensic investigation process 7 steps (determine goals/outline data locations/document and review plan/acquire and preserve/initial analysis/deeper investigation/report), drive image acquisition (write blockers — hardware vs software/forensic drive duplicators/dd command/dcfldd), image verification via hashing (SHA-256/MD5 — document both original and copy hash values), log acquisition steps (location/time period/obtain with documentation/validate with checksums/identify items of interest/analyze with tools), log analysis tools (Splunk/ELA/Sawmill/text editors/grep-awk-sed), target evidence locations (system/application/auth/network device/email/web/database/cloud logs), forensic toolkit components per NIST (digital forensic workstation/forensic investigation suite: Autopsy-FTK-EnCase/write blockers/forensic drive duplicators/wiped media/cables and adapters/camera/labels/notebooks/portable printer), post-incident RCA and lessons learned (blameless/action items/owners/deadlines/follow-up)
